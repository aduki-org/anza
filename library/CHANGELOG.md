## [0.2.1] ‚Äî 2026-06-09

### Added

#### Service Worker Toolkit ‚Äî `@adukiorg/anza/sw`

- **New subpath export** `@adukiorg/anza/sw` ‚Äî caching strategies, route interception, background sync, and push notifications
- **Seven caching strategies**: CacheFirst, NetworkFirst, StaleRevalidate, CacheThenNetwork, NetworkOnly, CacheOnly, OfflineFallback
- **URLPattern routing** inside the Service Worker via `router()` and `Router`
- **Install/activate helpers**: `precache()`, `pruneStale()`, `claim()`, `enableNavPreload()`
- **TTL expiry**: `pruneExpired()`, `setupAutoPrune()` with `x-expires-at` headers
- **Background sync**: `replayQueue()`, `requeueFailed()` with dead-letter queue
- **Request serialization**: `serializeRequest()`, `deserializeRequest()` for IndexedDB storage
- **Push notifications**: `subscribe()`, `notify()` with VAPID support
- **Scaffolded `src/sw.js`** generated by both `anza create` and `anza-create`

#### Theme Switching ‚Äî `@adukiorg/anza/theme`

- **New subpath export** `@adukiorg/anza/theme`
- **Auto-init on import** ‚Äî reads saved preference from `localStorage` or respects `prefers-color-scheme`
- **Attaches to `window.theme`** via `Object.defineProperty` (non-enumerable, non-configurable)
- **API**: `theme.get()`, `theme.set()`, `theme.toggle()` ‚Äî all update the same global instance
- **No manual init required** ‚Äî importing `@adukiorg/anza/ui` triggers it automatically

#### View Transition Tokens ‚Äî `tokens/semantic/transitions.css`

- **New semantic token layer** connecting the CSS View Transitions API to the design token system
- **Tokens**: `--transition-bg`, `--transition-duration`, `--transition-easing`, `--transition-push`, `--transition-pop`, `--transition-replace`
- **Router `transitions.run()`** injects a token-aware stylesheet on first use
- **Dock `swap()`** reads directional easing from tokens (`--transition-push` vs `--transition-pop`) for physically different forward/back feel
- **High-contrast theme morphing** ‚Äî `contrast.css` now declares `transition:` for smooth token interpolation

#### Documentation

- **Comprehensive `docs/index.md`** ‚Äî master docs entry with all modules listed by usefulness
- **`docs/sw/`** ‚Äî complete SW documentation: `index.md`, `start.md`, `strategies.md`, `routes.md`, `sync.md`, `api.md`
- **Updated `docs/intro/`** ‚Äî SW toolkit in feature list, build output includes `dist/sw.js`
- **View Transition token docs** added to `docs/ui/transitions.md`, `docs/router/transitions.md`, `docs/animations/tokens.md`

#### Tooling

- **`tools/src/build/graph.rs` split** into focused modules:
  - `cache.rs` ‚Äî incremental build cache (`.anzacache.json`)
  - `parse.rs` ‚Äî ESM AST parsing with swc
  - `entries.rs` ‚Äî entry point discovery from HTML and `src/sw.js`
  - `resolve.rs` ‚Äî import map and filesystem resolution
  - `html.rs` ‚Äî HTML injection (importmap link, HMR script)
  - `sw.rs` ‚Äî Service Worker bare-specifier rewriting
- **Auto-discovery of `src/sw.js`** as a build entry point
- **Bare specifier rewriting in `dist/sw.js`** ‚Äî `@adukiorg/anza/sw` ‚Üí `./sw/index.js` (SW does not support import maps)

### Changed

- **Cache file renamed** from `.anza-build-cache.json` to `.anzacache.json`
- **Scaffolded `src/app.js`** now imports and auto-registers the Service Worker
- **Scaffolded `src/sw.js`** included in both Node.js and Rust `create` commands

### Fixed

- High-contrast theme (`contrast.css`) now animates token changes instead of snapping instantly

## [0.2.0] ‚Äî 2026-06-07

### Added

#### Compiler ‚Äî `anza`

- **Dependency Graph Walking**: Automated dependency resolution of ESM imports/exports.
- **Tree-Shaking**: Outputs only referenced files to `dist/`, excluding unused code.
- **Automatic Inline Importmaps**: Optimal `<script type="importmap">` generated and injected inline into HTML entry points.
- **Relative Component Styles & Templates**: Parses relative template and stylesheet file paths referenced in `ui.element` relative to `import.meta.url`.
- **Concurrent File Watcher**: Recompiles output on file modifications.
- **SSE HMR Dev Server**: CSS hot-swapping and HTML/JS auto-reloading via Server-Sent Events (SSE).
- **Nested /dist Routing**: Added router nested service mapping in Axum dev server for development path parity.

#### Structure

- **Monorepo Split**: Reorganized files into `/library` (NPM package), `/sample` (demo application), and `/tools` (Rust compiler).

#### Sample

- **Cyberpunk Blog SPA**: Features reactive store, category filters, instant search, likes count persistence, details page with lightweight custom markdown renderer, post publishing, and post deletion.

## [0.1.0] ‚Äî 2026-05-27

### Added

#### Package

- Published as `@adukiorg/anza` ‚Äî pure browser ESM, zero build step
- Scoped subpath exports for every core module (`/api`, `/state`, `/storage`, etc.)
- `"type": "module"` ‚Äî fully native ESM, no CommonJS wrapper
- `npm test` via `@web/test-runner` (real Chromium, no jsdom)
- `npm run serve` via `@web/dev-server` on port 8080

#### Core ‚Äî `@adukiorg/anza/api`

- `execute()` ‚Äî fetch wrapper with AbortSignal, timeout, and `scheduler.postTask` integration
- `PlatformError` ‚Äî unified error shape across all network failures
- `retry()` ‚Äî exponential backoff with jitter and AbortSignal support
- `stream()` ‚Äî async generator streaming over NDJSON responses
- `createNDJSONTransform()` ‚Äî reusable `TransformStream` for NDJSON parsing
- `upload()` ‚Äî multipart file upload with progress events
- `pipeline` ‚Äî composable request/response middleware pipeline
- Cache strategies: `cache-first`, `network-first`, `stale-while-revalidate`

#### Core ‚Äî `@adukiorg/anza/state`

- `ReactiveStore` ‚Äî Proxy-based reactive state with microtask-batched notifications
- `setActiveSubscriber` / `getActiveSubscriber` ‚Äî dependency tracking context
- `derived()` ‚Äî auto-tracked computed values that re-evaluate on dependency changes
- `sync()` ‚Äî BroadcastChannel cross-tab state synchronization

#### Core ‚Äî `@adukiorg/anza/events`

- `EventBus` ‚Äî typed pub/sub with wildcard patterns and AbortSignal cleanup
- `events` ‚Äî singleton global event bus instance

#### Core ‚Äî `@adukiorg/anza/router`

- `register()` / `match()` ‚Äî URL pattern registration and matching
- `clear()` / `getRoutes()` ‚Äî route registry management
- `addGuard()` ‚Äî async navigation guard hooks
- `setNotFound()` ‚Äî 404 handler
- `setup()` ‚Äî bootstraps native Navigation API interception
- Full programmatic history API: `navigate`, `replace`, `back`, `forward`, `go`, `current`, `entries`
- `renderOutlet()` ‚Äî declarative route outlet rendering

#### Core ‚Äî `@adukiorg/anza/storage`

- `Database` ‚Äî Promise-wrapped IndexedDB with sequential migrations
- `LRUCache` / `WeakLRUCache` ‚Äî in-memory LRU caches with optional TTL
- `storage` ‚Äî unified tiered facade: memory ‚Üí IndexedDB ‚Üí Cache API ‚Üí OPFS
- `quota` ‚Äî storage estimate and persistence request helpers

#### Core ‚Äî `@adukiorg/anza/offline`

- `queue` ‚Äî IndexedDB-backed offline operation queue with FIFO dequeue
- `check()` / `subscribe()` ‚Äî connectivity detection and change subscriptions

#### Core ‚Äî `@adukiorg/anza/animations`

- `animate()` ‚Äî WAAPI wrapper with AbortSignal and easing controls
- `stagger()` ‚Äî staggered multi-element animation groups with `finished` promise

#### Core ‚Äî `@adukiorg/anza/workers`

- `lock()` ‚Äî Web Locks API facade with timeout and AbortSignal support
- `WorkerPool` ‚Äî managed pool of Web Workers with task queue and concurrency limits

#### Core ‚Äî `@adukiorg/anza/security`

- `sanitize()` ‚Äî XSS-safe HTML sanitizer using `DOMParser`
- `uuid()` ‚Äî `crypto.randomUUID()` wrapper
- `hash()` ‚Äî SHA-256/384/512 via Web Crypto API
- `generateKey()` / `deriveKey()` ‚Äî AES-GCM key generation and PBKDF2 derivation
- `encrypt()` / `decrypt()` ‚Äî AES-GCM symmetric encryption/decryption

#### Core ‚Äî `@adukiorg/anza/platform`

- `supports` ‚Äî feature detection registry for 30+ browser APIs
- `reset()` ‚Äî cache reset utility (used in tests)

#### Core ‚Äî `@adukiorg/anza/ui`

- `BaseElement` ‚Äî Shadow DOM base class for all custom elements
- Design token cascade: primitive ‚Üí semantic ‚Üí component token layers

#### Tests

- 26 test suites, 70 assertions ‚Äî all running in real Chromium via `@web/test-runner`
- Browser-native import maps injected per test run ‚Äî no Node.js module resolution

#### Blog Demo

- `blog/` ‚Äî sample SPA demonstrating state, storage, offline queue, and animations
- Import map mirrors the published `@adukiorg/anza/*` subpath exports exactly
